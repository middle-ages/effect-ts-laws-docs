Function testTypeclassLaws

Test typeclass laws for the given instances of some data type. This is just like testTypeclassLawsFor, but with all functions monomorphic on an underlying type of readonly number[].

  • testTypeclassLaws<F, In1, Out2, Out1>(instances, options, parameters?): void

  • Type Parameters

    • F extends TypeLambda
    • In1 = never
    • Out2 = unknown
    • Out1 = unknown

    Parameters

    • instances: Partial<ConcreteInstances<Kind<F, In1, Out2, Out1, Mono>> & ParameterizedInstances<F, In1, Out2, Out1>>

      Instances to test. Key is typeclass name and value is the instance under test. For example, { Monad: Option.Monad } will run the monad typeclass laws on Option.

    • options: MonomorphicOptions<F, In1, Out2, Out1>

      Type-specific test options. An object with _two fields which you must fill with the values specific to the data type under test:

      1. getEquivalence - A function that will build an Equivalence for your data type from an Equivalence for the underlying type.
      2. getArbitrary - A function that will build an Arbitrary for your data type from an Arbitrary for the underlying type.

      Optional runtime fc-check parameters.
